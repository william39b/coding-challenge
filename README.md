Her kommer ukas nøtt:

Denne uka er det en mer klassisk "nøtt", da den krever litt tenking og ren koding. Oppgaven er
delt inn i tre deler, fra enkleste (men ikke lett) til mest utfordrende:

Skriv en funksjon som returnerer,
-1 hvis input er negativt,
0 hvis input er 0,
1 hvis input er positivt.,
Du kan ikke bruke if/else eller ternary operators (x ? y : z).

Skriv ut tall fra 1 til N, men med ekstra regler:,
Delbart på 3 -> "Fizz",
Delbart på 5 -> "Buzz",
Delbart på begge -> "FizzBuzz",
Inneholder tallet 3 -> "Fizz" (f.eks. 13 -> "Fizz"),
Inneholder tallet 5 -> "Buzz" (f.eks. 52 -> "Buzz"),
1, 2, Fizz, 4, Buzz, Fizz, 7, 8, Fizz, Buzz, 11, Fizz, Fizz, 14, FizzBuzz, 16, 17, Fizz, 19, Buzz, Fizz, 22, Fizz, Fizz, Buzz, ...


Du har 12 mynter der én er falsk - den veier enten litt mindre eller litt mer enn de ekte. Bruk en balansevekt (en vekt som sammenligner to grupper mynter og viser om venstre side er tyngre, høyre side er tyngre, eller om de er like). Finn den falske mynten (og om den er lettere eller tyngre) ved å bruke veiingen (jeg trippelsjekka at veiing er et ekte ord, det er faktisk det, selv om det er stygt) maks 3 ganger.,
